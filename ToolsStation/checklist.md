# Checklist

## 代码质量
- [ ] 所有 Python 代码符合 PEP 8 规范 (使用 Black/Ruff 格式化).
- [ ] 前端代码无 ESLint 错误.
- [ ] 关键业务逻辑 (爬虫解析、数据处理) 有单元测试覆盖.
- [ ] 无硬编码的密钥或敏感信息 (使用环境变量).

## 功能验收
- [ ] 能够成功从至少 3 个不同来源抓取数据.
- [ ] 抓取的数据能够正确存储到数据库.
- [ ] LLM 能够生成可读的中文摘要和正确的分类.
- [ ] API 响应时间在合理范围内 (< 500ms for list, < 100ms for detail).
- [ ] 前端能够正确显示当天的资讯，且分类清晰.
- [ ] 移动端适配良好 (响应式布局).

## 稳定性与健壮性
- [ ] 爬虫遇到网络错误或反爬限制时有重试机制或优雅失败.
- [ ] LLM API 调用失败时有降级处理 (如仅展示原文标题).
- [ ] 定时任务能够自动恢复或记录错误日志.
